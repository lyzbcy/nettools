## v4.3 — 2026-07-22（免密授权）

**主题：解决"每操作弹密码"+"守护无法无人值守"两个用户痛点**

### 背景
- 每个特权操作（清 DNS、换 DNS、清代理、续租、VPN 共存）都独立弹一次系统密码框
- 守护每 30s 触发修复也会弹密码框，与"人不在电脑前保活远程连接"的设计目标完全矛盾

### 方案：setuid root helper
- 新增 `src/NetRepairHelper.swift`：编译成二进制，setuid root 后免密执行特权命令
- 固定 action 白名单（7 个特权操作），拒绝任意命令（安全约束）
- 参数严格校验（IP/网卡名正则），防注入
- macOS 默认忽略 shell 脚本 setuid（`kern.sugid_scripts=0`），所以 helper 必须是编译二进制

### 改动
- **新增** `src/NetRepairHelper.swift`（setuid 助手，~280 行）
- **改** `src/NetRepair.swift`：
  - `runAsAdmin` → `runAsAdminOSA`（仅用于安装/卸载 helper）
  - 新增 `runPrivileged`：优先走 helper 免密，未装则 fallback osascript
  - 7 个特权函数全部改走 `runPrivileged`
  - 新增 action：`helper-status` / `install-helper` / `uninstall-helper`
- **改** `build.sh`：加 helper 编译步骤，helper 打包进 app Resources
- **改** `index.html`：修复工具页加「免密授权」卡片，守护文案随授权状态联动
- **改** `doc/AI-CONTEXT.md`：更新为 v4.3 架构

### 用户体验
- 点「安装免密授权」→ 输一次开机密码 → 之后永久免密（重启不影响）
- 所有特权操作 + 守护都不再弹密码框
- 守护真正可无人值守保活远程连接
- 可随时「卸载授权」恢复弹密码模式

### 验证
- helper 编译通过，self-test 正常，白名单校验生效
- 主程序编译通过，18 个 action 全部接通
- ad-hoc 签名通过，app 正常启动
- 待用户实测：安装授权 → 验证免密 → 开守护验证无人值守

## v4.2 — 2026-07-17（执行层重构）

**主题：去掉花架子，按真实故障对症处理**

### 执行层

- 从旧 Mach-O 行为重新建立 `src/NetRepair.swift`，恢复可维护源码
- 新增 `build.sh`，可重复编译、签名和验证
- JS 桥接增加 35 秒/150 秒超时，不再永久 pending
- 新增只读 `--self-test` 与单 action 自检入口

### 网络诊断与修复

- 综合使用网卡/IP、原始 IP、DNS、HTTPS 和代理端口，不再只凭 ping 判定断网
- 修复 `networksetup` 错把 `en0` 当网络服务名的问题
- 清代理改为“备份后临时关闭当前服务”，不再删除 Git/npm 配置；新增恢复入口
- DHCP 只续租当前物理接口，不再关闭/打开 WiFi
- WiFi 改为解析 `system_profiler` JSON，适配 macOS 26
- 守护按 issue 对症修复；代理正常监听时绝不关闭

### 公司 VPN 共存

- 先补公司网段和公司 DNS 的精确 VPN 路由，再删除半世界路由
- 写入 `/etc/resolver/<公司域名>`，`<公司域名>` 单独查询公司 DNS
- 代理绕过加入 `*.<公司域名>` / `<内网域名>`
- 检测并清理旧版遗留的全局公司 DNS，其他 DNS 与 Tailscale 保持不变
- 新增共存配置恢复入口与四项结果验证

### 纠正历史误判

- `<内网域名>` 的正确目标是公司内网 `<内网前缀>.*`，不是 Oracle Cloud 公网服务
- `<公网错误IP>` 是公共/Tailscale DNS 的错误答案
- 不能关闭 Tailscale DNS，也不能把公司 DNS 设为全局；正确方法是分域 DNS + 精确路由 + 代理直连

## v4.1.1 — 2026-07-17（DNS 修正补丁）

**历史说明：这一版的判断后来被真实对照测试推翻，保留记录但不得继续沿用。**

### 背景
当时误以为 `<内网域名>` 是 Oracle Cloud 公网服务，并尝试在共存后恢复自动 DNS。后续对照证明：同事可用域名、用户可用 `<内网探针IP>`，Tailscale DNS 则返回 `<公网错误IP>`；正确目标实际是公司内网地址。

### 修复
- 该版只处理了全局 DNS 副作用，没有完成 `<公司域名>` 分域解析，也没有把 `*.<公司域名>` 加入代理直连，因此并未真正解决共存。
- v4.2 已替代此实现。

### 验证
- 本节原“验证”只证明了公共 DNS 会返回错误地址，不能证明该公网地址是正确 GitLab。

### 同时修复
- `runDiagnose` 的 `querySelector('')` 崩溃（从仪表盘点诊断会抛异常，v4.0 原始 bug）
- 清理 `guardian-nextcheck` 死代码
